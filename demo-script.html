🎬 现场演示脚本

预计总时长：15-20 分钟 · 演示 + 讲解 + Q&A
1

开场：为什么需要多 Agent

2 分钟
打开 PPT 第 1-2 页
大家好，今天分享一下我用 OpenClaw 搭建的多 Agent 协作方案。先说下背景——我们平时用 ChatGPT 或 Kimi 都是单一对话框，上下文一长就容易丢信息，而且没有专业化分工。
我的想法是：像管理真实团队一样管理 AI。让不同的 AI 扮演不同的角色，各司其职。
2

架构介绍：4 人 AI 团队

3 分钟
PPT 第 3 页（架构图）
整个团队由 4 个 AI Agent 组成：CEO 负责统筹，下面是商务、运营、产研三条线。我只需要跟 CEO 说话，它会自动拆解任务、分派给三条线、最后汇总结果返回。
PPT 第 5 页（SOUL 灵魂）
关键在于每个 Agent 都有自己的"灵魂定义"——SOUL 文件。它决定了 Agent 是谁、擅长什么、跟谁协作。SOUL 越具体，回答越专业。
如果老板问"跟直接用 ChatGPT 有什么区别"，重点强调：专业化分工 + 并行处理 + 知识边界防幻觉
3

现场演示：给 CEO 下任务

5-8 分钟
打开手机 → CEO Bot 对话
现在我直接演示。我打开 CEO Bot 的对话窗口，给它发一条消息。
发送消息："帮我分析一下 Q2 华为渠道的合作策略，从商务、运营、产研三个角度给建议"
这是一个跨线需求。CEO 收到后会拆解成三个子任务，分别派给三条线的负责人。
等待 CEO 回复（约 30-60 秒）
大家注意看，CEO 先告诉我它在协调三条线，然后三条线并行执行。传统方式你得分别找三个人问，再自己汇总。这里 CEO 会自动整合三方结果。
如果等待时间较长，可以切到 Dashboard（http://127.0.0.1:18789）展示 session 列表，让老板看到多个 Agent 在并行工作
展示 CEO 汇总结果
看，CEO 把商务、运营、产研三条线的结果整合成了一份完整的方案，结构清晰，各线职责明确。这就是多 Agent 协作的价值——你只说一句话，团队就跑起来了。
（可选）直接跟商务负责人对话
切到商务负责人 Bot，发送："华为渠道最近的合作政策有什么变化？CPS 分成比例一般怎么谈？"
也可以直接找某个负责人聊特定领域的问题，它会用商务的专业视角回答。
4

技术亮点 & 防幻觉设计

2 分钟
PPT 第 6 页（设计理念）
几个关键设计：一是每个 SOUL 都声明了知识边界，遇到不知道的会标注"待确认"，不会乱编数据。二是 CEO 有汇总原则，收齐三条线再输出，不会零散转发。三是一键部署，同事 10 分钟就能跑起来。

成果展示 & 可扩展性

2 分钟
PPT 第 7-8 页
目前 4 个 Agent 已经稳定运行。架构设计上支持 3 级扩展，人数按业务需求灵活配置。而且这套配置可以复用到电商、内容创业、技术团队等不同场景——只需要改 SOUL 文件。
PPT 第 9 页（下一步）
下一步计划：推广给组内同事使用，接入内部知识库让 Agent 回答更精准，根据反馈持续迭代 SOUL。项目已经开源到 GitHub。
6

Q&A 预案

3-5 分钟
Q: 这跟直接用 ChatGPT 有啥区别？
三个核心区别：第一，专业化分工，每个 Agent 只做擅长的事；第二，并行处理，三条线同时跑；第三，知识边界，不会乱编数据。
Q: API 费用贵吗？
4 人团队日常使用大约每天 5-15 块，用的是 Moonshot Kimi 模型。可以根据使用频率调整。
Q: 安全性怎么样？敏感信息会不会泄露？
所有数据都在本地跑，不经过第三方服务器。对外开源的配置也做了脱敏处理，API Key 等全部替换成占位符。
Q: 其他业务线能用吗？
可以，只需要改 SOUL 文件就行。架构是通用的，换个行业就换一套角色定义。
Q: 会不会影响正常工作？
不会，它是辅助工具。帮你快速出方案初稿、做结构化分析，最终决策还是人来做。
